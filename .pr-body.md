Per Tori (confirmed by Javier Rengel), the iframe embed route does not include a `/dashboard/{dashboardUuid}` (or slug) segment — using it redirects users to the login page. The `dashboardUuid` is passed inside the JWT payload as `content.dashboardUuid`.

Changes to `guides/embedding/dashboards.mdx`:
- URL structure section: replaced with `https://your-instance.lightdash.cloud/embed/{projectUuid}#{jwtToken}` and removed the dashboard slug variant
- JS, Python, and Ruby `embed_url` examples updated to drop the `/dashboard/...` segment
- HTML `<iframe>` src and the theming example URL updated for consistency on the same page

Out of scope (not changed): `references/iframe-embedding.mdx` and `references/embedding.mdx` still contain the old URL pattern. Happy to follow up if you want those fixed too.
