Every board deck has a pie chart. This is the number behind it: each segment's share of the whole.

<Info>**Status:** 🟡 Supported (Experimental) — use the `percent_of_total` metric type, or fall back to a table calculation.</Info>

## The question

> "What percentage of total revenue does each product category represent?"

## Why it matters

Absolute numbers hide structural shifts. Revenue might grow 20% overall, but if one category quietly climbs from 15% to 35% of the mix, you have a concentration risk — or an emerging star worth doubling down on. Mix analysis turns raw totals into strategic insight.

## Example

An e-commerce store selling across five product categories last quarter.

### Source data

| order_id | category | order_total |
|----------|-------------|-------------|
| 3001 | Electronics | 249.00 |
| 3002 | Clothing | 67.00 |
| 3003 | Home & Garden | 112.00 |
| 3004 | Electronics | 189.00 |
| 3005 | Books | 34.00 |
| 3006 | Sports | 85.00 |
| 3007 | Clothing | 93.00 |
| 3008 | Electronics | 310.00 |

### YAML

```yaml
version: 2
models:
  - name: orders
    columns:
      - name: category
        meta:
          dimension:
            type: string
      - name: order_total
        meta:
          metrics:
            total_revenue:
              type: sum
              format: "usd"
            revenue_mix:
              type: percent_of_total
              sql: "order_total"
              format: "percent"
```

If `percent_of_total` is not yet enabled in your Lightdash version, add a table calculation instead:

```
${orders.total_revenue} / SUM(${orders.total_revenue}) OVER ()
```

### Result

| category | total_revenue | revenue_mix |
|-------------|---------------|-------------|
| Electronics | $148,200 | 42.0% |
| Clothing | $81,100 | 23.0% |
| Home & Garden | $56,300 | 16.0% |
| Sports | $42,200 | 12.0% |
| Books | $24,700 | 7.0% |

## How to set it up

1. Define a `sum` metric on your revenue column (e.g. `total_revenue`).
2. Add a second metric with `type: percent_of_total` on the same column, or create a table calculation using the window function `SUM(...) OVER ()`.
3. Select your segment dimension (e.g. `category`) and both metrics in the Explore view.
4. Sort by `revenue_mix` descending to see the dominant segments first.
5. Format as a stacked bar or pie chart on your dashboard.

## Tips

- Combine with a time dimension to watch how the mix shifts quarter over quarter.
- If you filter the query (e.g. to one region), the percentages automatically re-base to that subset's total.
