Sort, cut, number. The leaderboard is the simplest way to focus attention on what matters most.

<Info>**Status:** ✅ Fully supported for simple Top N. Top N per group (e.g. top 3 products per region) requires a `PARTITION BY` table calculation.</Info>

## The question

> "Who are our top 10 customers by lifetime spend?"

## Why it matters

Pareto rules most businesses: a small share of customers drives a large share of revenue. A leaderboard makes that concentration visible and actionable — protect these accounts, model look-alikes, flag churn risk. Lightdash handles this natively with sort and row limits.

### YAML

```yaml
version: 2
models:
  - name: customers
    columns:
      - name: customer_name
        meta:
          dimension:
            type: string
      - name: total_spend
        meta:
          metrics:
            lifetime_spend:
              type: sum
              format: "usd"
```

Add a table calculation for the rank column:

```
ROW_NUMBER() OVER (ORDER BY ${customers.lifetime_spend} DESC)
```

## How to set it up

1. Define a dimension for the entity you're ranking (e.g. `customer_name`) and a metric to rank by (e.g. `lifetime_spend`).
2. In the Explore view, select both, then **sort by the metric descending**.
3. Set the **row limit to 10** (or your desired N) in the query settings.
4. Add a table calculation with `ROW_NUMBER() OVER (ORDER BY ${customers.lifetime_spend} DESC)` to display explicit rank numbers.
5. Save as a table chart or horizontal bar chart on your dashboard.

## Top N per group

For "top 3 products per region" style queries, you need a partitioned rank:

```
ROW_NUMBER() OVER (PARTITION BY ${orders.region} ORDER BY ${orders.total_revenue} DESC)
```

Then filter the table calculation to keep only rows where `rank <= 3`. This works but gets unwieldy with many groups. If you hit that wall, pre-compute the rank in a dbt model:

```sql
-- models/top_products_by_region.sql
SELECT *
FROM (
  SELECT
    region,
    product_name,
    revenue,
    ROW_NUMBER() OVER (
      PARTITION BY region ORDER BY revenue DESC
    ) AS rank
  FROM {{ ref('product_sales') }}
)
WHERE rank <= 3
```
