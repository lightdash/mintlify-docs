Every analytics question starts the same way: take many rows, group them, and reduce each group to a single number.

<Info>**Status:** ✅ Fully supported — this is bread-and-butter Lightdash.</Info>

## The question

> "What's our total revenue by country?"

## Why it matters

Without roll-ups, you're staring at thousands of individual order rows. Grouping turns raw transactions into decisions: which market to invest in, which product to cut, where to hire. Lightdash generates the `group by` and aggregation from your YAML definitions, so analysts never write SQL for this.

## Example

An online store with orders across five countries.

### Source data

| order_id | country | order_total |
|----------|---------|-------------|
| 1001 | United States | 89.00 |
| 1002 | United Kingdom | 134.50 |
| 1003 | Germany | 67.00 |
| 1004 | United States | 212.00 |
| 1005 | Canada | 45.00 |
| 1006 | Australia | 178.00 |
| 1007 | Germany | 92.00 |
| 1008 | United States | 56.00 |

### YAML

```yaml
version: 2
models:
  - name: orders
    meta:
      joins: []
    columns:
      - name: country
        meta:
          dimension:
            type: string
      - name: order_total
        meta:
          metrics:
            total_revenue:
              type: sum
              format: "usd"
```

### Result

| country | total_revenue |
|---------|---------------|
| United States | $142,380 |
| United Kingdom | $89,210 |
| Germany | $73,540 |
| Canada | $51,890 |
| Australia | $38,720 |

## How to set it up

1. Define a dimension on the column you want to group by (e.g. `country`).
2. Define a metric with an aggregation type (`sum`, `count`, `average`, `min`, `max`) on the value column.
3. Open the Explore view, select the dimension and the metric, and hit **Run query**.
4. Lightdash generates `select country, sum(order_total) ... group by country` behind the scenes.
5. Sort, chart, or save to a dashboard.

## Tips

- Add `format: "usd"` or `round: 2` to your metric for clean display.
- Multiple dimensions work — add `product_category` for a two-level grouping.
