# Basic Query Patterns

The bread-and-butter patterns that cover most day-to-day analysis. If you're new to Lightdash, start here.

---

## 1. The Roll-Up

**What you're trying to do:** Group your data by a category and get a total, average, or count for each group.

**Example question:** *"What's our total revenue by country?"*

**Lightdash features used:**
- Pick a dimension (Country) and a metric (Total Revenue) in the Explore view
- Lightdash generates the `group by` and aggregation for you

**Status:** ✅ Fully supported — this is bread-and-butter Lightdash.

---

## 2. The Lookup

**What you're trying to do:** Pull in a column from a related table. You have orders, but you want to see the customer's plan tier next to each order.

**Example question:** *"Show me order revenue with each customer's plan tier."*

**Lightdash features used:**
- Joins defined in your YAML model (e.g. orders → customers)
- Select dimensions from the joined table alongside your base metrics

**Status:** ✅ Fully supported — Lightdash's join system handles M:1 lookups cleanly.

---

## 3. The Filter Drill

**What you're trying to do:** Start broad, then drill down into specifics. Click on a region to see cities, click a city to see individual reps.

**Example question:** *"Revenue by region → just EMEA → by city → by rep."*

**Lightdash features used:**
- Dashboard cross-filtering (click a chart value to filter all tiles)
- Dimension filters with AND/OR logic
- Drill into underlying data from any chart

**Status:** ✅ Fully supported.

---

## 4. The Conditional Metric

**What you're trying to do:** Count or sum only the rows that match a condition. Like counting only paid users, or summing revenue from just one product line.

**Example question:** *"How many users signed up last month who are on the Pro plan?"*

**Lightdash features used:**
- Metric filters (`filters:` in YAML) — e.g. a "Pro Users" metric that only counts users where `plan = 'pro'`
- Or use custom metrics in the Explore UI to add a filter on the fly

**Status:** ✅ Fully supported.

---

## 5. The Ratio

**What you're trying to do:** Divide one metric by another. Revenue per user, cost per acquisition, average order value.

**Example question:** *"What's our revenue per active user by month?"*

**Lightdash features used:**
- Derived metrics (`type: number`) that reference other metrics: `sql: ${total_revenue} / nullif(${active_users}, 0)`
- Both metrics are computed in the same query, then divided

**Status:** ✅ Fully supported — as long as both metrics are at the same grain (see The Benchmark for when they're not).

---

## 6. The Pivot

**What you're trying to do:** Turn row values into columns. Instead of one row per month, you want months as column headers with products down the side.

**Example question:** *"Show me revenue by product, with each month as a column."*

**Lightdash features used:**
- Pivot button in the results table — drag a dimension to columns
- Supports up to 3 pivot dimensions and 200 columns
- Subtotals and totals available

**Status:** ✅ Fully supported.
