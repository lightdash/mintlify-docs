Cohort analysis groups users by when they arrived, then tracks how many remain active as time passes.

<Info>**Status:** 🟡 Supported if your cohort model exists in dbt — the semantic layer doesn't generate cohort-based time offsets on its own.</Info>

## The question

> "What percentage of users who signed up in January are still active three months later?"

## Why it matters

Overall retention rates are misleading. A company with 70% monthly retention might actually be improving — January's cohort retained 60%, but March's retained 78%. Without cohort isolation, a growing user base masks a deteriorating (or improving) product. The retention triangle is the only honest view.

## Example

A SaaS app tracking monthly active users by signup cohort.

### Source data

One row per cohort-month combination, produced by a dbt model.

| cohort_month | months_since_signup | active_users |
|--------------|---------------------|--------------|
| 2025-01      | 0                   | 1,200        |
| 2025-01      | 1                   | 840          |
| 2025-01      | 2                   | 620          |
| 2025-01      | 3                   | 510          |
| 2025-02      | 0                   | 1,450        |
| 2025-02      | 1                   | 1,015        |
| 2025-02      | 2                   | 770          |
| 2025-03      | 0                   | 1,680        |
| 2025-03      | 1                   | 1,210        |
| 2025-04      | 0                   | 1,920        |

### YAML

```yaml
version: 2
models:
  - name: cohort_retention
    meta:
      joins: []
    columns:
      - name: cohort_month
        meta:
          dimension:
            type: date
            time_intervals:
              - month
      - name: months_since_signup
        meta:
          dimension:
            type: number
      - name: active_users
        meta:
          metrics:
            total_active_users:
              type: sum
      - name: cohort_size
        description: "Users at month 0 — joined via a window function in dbt"
        meta:
          dimension:
            type: number
          metrics:
            retention_rate:
              type: number
              sql: "${total_active_users} / nullif(${cohort_size}, 0)"
              format: "percent"
```

### Result — the retention triangle

Pivot `cohort_month` into rows and `months_since_signup` into columns:

## How to set it up

1. Build a dbt model that creates the cohort grid (see workaround below).
2. Define `cohort_month` and `months_since_signup` as **dimensions**.
3. Define `total_active_users` as a **sum metric** and `retention_rate` as a **ratio metric** dividing active users by cohort size.
4. In Lightdash, select `cohort_month`, `months_since_signup`, and `retention_rate`. Use the **Pivot** button to move `months_since_signup` to columns.
5. Apply a `format: "percent"` to display clean percentages.

## Workaround

Build a dbt model that computes the cohort grid from raw data:

```sql
with user_cohorts as (
  select
    user_id,
    date_trunc('month', created_at) as cohort_month
  from {{ ref('users') }}
),

monthly_activity as (
  select
    u.user_id,
    u.cohort_month,
    datediff('month', u.cohort_month, date_trunc('month', e.activity_date))
      as months_since_signup,
    count(distinct u.user_id) as active_users
  from user_cohorts u
  inner join {{ ref('events') }} e on u.user_id = e.user_id
  group by 1, 2, 3
)

select
  cohort_month,
  months_since_signup,
  count(distinct user_id) as active_users,
  max(count(distinct user_id)) over (partition by cohort_month) as cohort_size
from monthly_activity
where months_since_signup >= 0
group by 1, 2
```

Point your Lightdash YAML at this model. The retention triangle appears in the Explore view with a single pivot.
