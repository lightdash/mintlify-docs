Sankey diagrams visualize flows between categories — how values move from a source to a target, weighted by a metric. They're ideal for understanding movement, distribution, or conversion across stages.

Sankey charts are useful when:

- You want to show how values flow between categories (e.g., marketing channels → conversions)
- You're visualizing multi-step processes or funnels
- You need to understand distribution across interconnected stages
- You have cyclical flows that loop back (e.g., retargeting → conversion → retargeting)

## Data requirements

Sankey charts require **three columns** in your data:

| Column type | Description | Example |
|-------------|-------------|---------|
| **Source dimension** | Where the flow originates | `channel`, `referrer`, `stage_from` |
| **Target dimension** | Where the flow goes | `conversion_type`, `destination`, `stage_to` |
| **Weight metric** | The size of the flow | `count`, `revenue`, `users` |

Each row in your data represents a flow from one node to another, with the weight determining the thickness of the connection.

## Creating a Sankey chart

1. Run a query with your source dimension, target dimension, and weight metric
2. Select **Sankey** from the chart type picker
3. In the configuration panel, assign:
   - **Source** — the dimension representing flow origins
   - **Target** — the dimension representing flow destinations  
   - **Value** — the metric determining flow size

## Display options

### Orientation

Display flows horizontally (left to right) or vertically (top to bottom).

### Node layout

Node layout controls how source and target labels are arranged across the chart. Choose the option that matches the story you want to tell.

#### Multi-step

The default layout. Nodes that appear as both a source and a target are chained into multi-level paths, so you can follow the full journey across stages.

For example, if "Email" appears as a target from "Ad Click" and as a source to "Purchase", the chart shows `Ad Click → Email → Purchase` as a connected path.

Multi-step also supports cyclical data where flows loop back. For example, in a retargeting funnel:

- Awareness → Consideration
- Consideration → Conversion
- Conversion → Retargeting
- Retargeting → Conversion (loop)

The chart handles these cycles by creating step-suffixed node instances (e.g., "Conversion - Step 2").

Use multi-step when you're visualizing funnels, journeys, or any process with three or more stages.

#### Merged

Like multi-step, but nodes with the same label across different stages are merged into a single node. This produces a cleaner view when you care about totals per category rather than the order of steps.

Merging requires an acyclic graph. If your data contains cycles, the Merged option is disabled and the chart falls back to Multi-step.

Use merged when you want a compact view that highlights overall volume per category.

#### Direct

Renders a strict two-column source → target view with no chaining. A label that is both a source and a target appears once on each side — for example, a left "Email" feeding in and a right "Email" being fed into — because source and target identities are kept separate.

Because the output is bipartite, Direct always renders successfully, including for cyclical data.

Use direct when you only want to see the immediate relationship between two categories and don't need to follow values across multiple steps.

### Node alignment

Align nodes to the left, right, or justify them across the chart. Node alignment applies to the **Multi-step** and **Merged** layouts; it's hidden for **Direct** because nodes are fixed to two columns.
