---
title: Semantic Layer Recipe Book
---

Every analysis you run in a BI tool follows one of a handful of patterns. This guide catalogs those patterns as **recipes** — showing you which ones Lightdash handles out of the box, which need workarounds, and what to do when you need something more.

Pick a category below to dive in, or scroll down for the full quick-reference table.

---

## Basics

The foundational query patterns you will use every day.

- [The Roll-Up](/semantic-layer/roll-up) — Aggregate measures grouped by dimensions
- [The Lookup](/semantic-layer/lookup) — Pull related attributes across tables via joins
- [The Filter Drill](/semantic-layer/filter-drill) — Narrow results with cross-filtering and nested filters
- [The Conditional Metric](/semantic-layer/conditional-metric) — Metrics that only count rows matching a condition
- [The Ratio](/semantic-layer/ratio) — Divide one metric by another using derived metrics
- [The Pivot](/semantic-layer/pivot) — Rotate dimension values into columns

## Time Series

Patterns for analyzing data across time periods.

- [The Period Comparison](/semantic-layer/period-comparison) — Compare metrics across time windows (e.g. MoM, YoY)
- [The Running Total](/semantic-layer/running-total) — Cumulative sums that grow over time
- [The Trailing Window](/semantic-layer/trailing-window) — Rolling averages and moving aggregates
- [The Snapshot](/semantic-layer/snapshot) — Point-in-time balances and inventory levels

## Proportions & Rankings

Patterns for understanding relative size and ordering.

- [The Mix](/semantic-layer/mix) — Percent-of-total breakdowns
- [The Benchmark](/semantic-layer/benchmark) — Compare individual values against a fixed aggregate
- [The Leaderboard](/semantic-layer/leaderboard) — Rank, sort, and top-N lists

## Events & Lifecycle

Patterns for tracking user journeys and retention over time.

- [The Funnel](/semantic-layer/funnel) — Multi-step conversion analysis
- [The Cohort](/semantic-layer/cohort) — Retention and lifecycle analysis by signup cohort

## Joins & Multi-Source

Patterns for combining data from multiple tables or sources.

- [The Fan-Out Trap](/semantic-layer/fan-out-trap) — Avoid double-counting with symmetric aggregates
- [The Blended Report](/semantic-layer/blended-report) — Merge queries from unrelated tables
- [The Big Wide Table](/semantic-layer/big-wide-table) — Replace denormalized wide tables with joins

## Dynamic & Governed

Patterns for parameterized analysis and access control.

- [Scenario Analysis](/semantic-layer/what-if) — Test assumptions with parameterised scenarios
- [Swappable Dimensions](/semantic-layer/dynamic-group) — Let users choose grouping dimensions at query time
- [Swappable Metrics](/semantic-layer/swappable-metrics) — Let users choose which measure to plot
- [Row-Level Security](/semantic-layer/restricted-view) — Filter data by who is viewing
